🤖 Sobre o Bot

O que é o Architect?

+

O Architect é um bot para Discord que usa Inteligência Artificial para criar servidores completos do zero. Além da criação, oferece sistema de backup, restauração após nukes, proteção anti-nuke em tempo real, reaction roles e muito mais.

O Architect é gratuito?

+

Sim! Todos os comandos do Architect são completamente gratuitos. Não há planos pagos, sem cobranças ocultas e sem limites de uso.

Quem desenvolveu o Architect?

+

O Architect foi desenvolvido pela Alzhayds, uma empresa brasileira focada em desenvolvimento de bots e ferramentas para Discord. A versão atual é a v1.5.0.

🏗️ Criação de Servidores

Como o /criar_servidor funciona?

+

Você usa o comando /criar_servidor e descreve em texto o servidor que quer. A IA analisa o prompt e cria automaticamente todos os cargos com cores e permissões, categorias, canais de texto, voz, fórum, announcement e palco — tudo organizado e profissional.

O /criar_servidor apaga meu servidor atual?

+

Sim! Antes de criar a nova estrutura, o bot apaga todos os canais e cargos existentes. Por isso aparece uma tela de confirmação antes de executar. Recomendamos usar /backup antes de usar o /criar_servidor.

O que são os Templates?

+

Os templates são estruturas prontas para tipos específicos de servidor. Use /template e escolha entre: Comunidade, Gaming, Militar, Loja, Anime ou Educacional. É a forma mais rápida de criar um servidor profissional sem precisar descrever tudo.

💾 Backup e Restauração

O que o /backup salva?

+

O /backup salva toda a estrutura do servidor: cargos com cores e permissões, categorias, canais (texto, voz, fórum, palco), permissões por canal, tópicos e as últimas 50 mensagens de cada canal de texto.

Meu servidor foi nukado, como restauro?

+

Se você tinha um backup salvo, use o comando /restaurar. O bot irá recriar toda a estrutura exatamente como estava, incluindo cargos, canais e mensagens salvas. Por isso é importante fazer backup regularmente!

Posso ter múltiplos backups?

+

Atualmente o Architect mantém 1 backup por servidor. Cada vez que você usa /backup, o backup anterior é substituído pelo novo. Planejamos adicionar múltiplos backups em versões futuras.

🛡️ Proteção Anti-Nuke

Como funciona a proteção anti-nuke?

+

O sistema monitora em tempo real as deleções de canais e cargos. Se detectar 3 ou mais deleções em menos de 10 segundos pelo mesmo usuário, envia um alerta imediato no servidor. Para ativar use /proteger ativo:true.

O bot bane o nukador automaticamente?

+

Não. O bot detecta e alerta sobre a atividade suspeita, mas o ban deve ser feito manualmente pelo dono do servidor. Isso evita bans falsos positivos.

Preciso de backup para ativar a proteção?

+

Sim! É necessário ter um backup salvo antes de ativar a proteção anti-nuke. Use /backup primeiro e depois /proteger ativo:true.

⚙️ Permissões e Configuração

Quais permissões o bot precisa?

+

O Architect precisa de permissão de Administrador para funcionar corretamente. Isso é necessário para criar e deletar canais, criar e deletar cargos, e monitorar o audit log para o sistema anti-nuke.

Qualquer membro pode usar os comandos?

+

Não. Os comandos principais (/criar_servidor, /backup, /restaurar, /proteger, /deletar, /cargo_criar, /canal_criar) requerem permissão de Administrador. Apenas /info, /help e /status são públicos para todos os membros.
